# ADR-001: Storage Key Design

**Status:** Accepted  
**Date:** 2026-07-01

## Context

We need a consistent, collision-resistant storage key scheme across all contracts generated by Forge.

## Decision

Use a typed Rust enum decorated with `#[contracttype]` as the storage key:

```rust
#[contracttype]
pub enum DataKey {
    Admin,
    Initialized,
    Pool(u32),
    Position(u32, Address),
    NextPoolId,
}
```

## Rationale

- Type safety: the compiler prevents accidental key reuse
- Namespaced by variant: `Pool(0)` and `Pool(1)` are distinct keys
- XDR-serialised: Soroban natively encodes enum variants with their discriminant

## Consequences

- All contracts must define a `DataKey` enum
- Forge code generator emits a default `DataKey` that templates extend
